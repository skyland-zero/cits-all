# 工单管理模块需求与设计文档

## 1. 模块概述
工单管理模块（WorkOrder Management）是 CITS 系统的核心业务模块，旨在实现设备报修、现场作业及服务请求的全生命周期数字化管理。系统采用“状态机驱动”的设计理念，确保业务流程的严谨性与可追溯性。

---

## 2. 现状分析 (已实现功能)

### 2.1 业务流程
目前系统已通过 `Stateless` 状态机实现以下闭环流程：
- **创建与提交**: 支持保存为 `Draft` (草稿) 或直接 `Submit` (提交) 至待分派池。
- **任务分派**: 管理员可将工单 `Assign` (分派) 给指定的处理人。
- **执行过程**: 处理人可执行 `Start` (开始) 和 `Finish` (完成上报) 操作。
- **审核结单**: 支持 `Approve` (通过并结单) 或 `Reject` (驳回重处理)。
- **操作审计**: 每一笔状态流转均自动记录 `WorkOrderLog`，包含操作人、动作、备注及变更前后的状态。

### 2.2 核心数据模型
- **工单主表**: 包含业务单号 (OrderNo)、优先级 (Priority)、故障描述、地理位置、附件 (JSON)、SLA 截止时间等。
- **流转日志**: 记录完整的业务生命周期轨迹。

### 2.3 技术特征
- **后端**: .NET 10 + FreeSql (ORM) + Stateless (状态机) + 单元事务处理。
- **前端**: Vue 3 + Element Plus (web-ele)，具备列表筛选、表单录入及状态视觉区分。

## 3. 扩展需求规划 (待开发功能)

### 3.1 消息通知集成 (高优先级)
- **企业微信对接**: 结合项目中已有的 `CorpWx` 模块，在以下节点发送通知：
    - 工单被分派时：通知处理人。
    - 处理完成时：通知审核员。
    - 工单被驳回时：通知原处理人。
- **超时预警**: 距离 `DeadlineTime` 不足 2 小时时发送催办提醒。

### 3.2 报表与防伪技术 (核心亮点)
- **PDF 报告生成**: 实现 `ProcessSecureReportAsync` 预留接口，根据工单内容生成正式的作业报告。
- **防伪数字指纹**: 
    - 在 PDF 生成时，提取工单关键数据（ID、操作人、时间）生成 Hash。
    - 将 Hash 作为隐形水印或数字指纹注入 PDF。
    - 提供“扫码核验”功能，验证报告的真实性。

### 3.3 SLA 与自动调度
- **自动逾期标记**: 后端挂载后台任务，定期检查 `DeadlineTime`，将超期未处理的工单标记为“已逾期”。
- **智能分派预留**: 基于处理人的当前在办数，提供推荐分派建议。

### 3.4 交互增强
- **评论与交互**: 在详情页增加“流转记录”之外的“互动备注”，支持多人针对某一工单进行讨论并上传补充照片。
- **移动端适配**: 为现场作业人员提供适配的 H5 或移动端视图（基于 `web-ele` 的移动适配或独立应用）。

### 3.5 多端同步
- **Ant Design / Naive UI 适配**: 将 `web-ele` 中已有的业务组件逻辑迁移至 `web-antd` 和 `web-naive`，确保 monorepo 中各 UI 版本功能对等。

## 4. 接口规范参考
| 动作 | 方法 | 路径 | 说明 |
| :--- | :--- | :--- | :--- |
| 提交 | POST | `/api/work-order/{id}/submit` | 草稿转待分派 |
| 分派 | POST | `/api/work-order/{id}/assign` | 指定处理人 |
| 驳回 | POST | `/api/work-order/{id}/reject` | 审核不通过，返回处理中 |
| 结单 | POST | `/api/work-order/{id}/approve` | 生成防伪报告并结束流程 |
